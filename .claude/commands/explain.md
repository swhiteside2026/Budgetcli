Ask the user which file they would like explained.

Once they provide a file path, read the file and explain every function in plain English. For each function include:

- What it does in one or two sentences
- What it takes as input (each parameter, its type, and what it represents)
- What it returns (type and meaning)
- How it connects to the rest of the project (which functions call it, which it calls, and why it matters)

Keep the explanations clear and jargon-free. Structure the output with one section per function in the order they appear in the file.
